Mini Shell

C Linux Systems Programming

Overview

A fully functional Linux shell implementation with advanced features for process management and I/O handling.

Key Features

  • Script execution with I/O redirection (stdin, stdout, stderr)
  • Support for multiple piped commands
  • Background and foreground process execution
  • Signal handling for program interruption (SIGINT, SIGTSTP)
  • Command history with readline library integration
  • File-lock detection using procfs
  • Malware detection algorithm based on file locks

Technical Implementation

Built using C system calls (fork, exec, pipe, dup2) and signal handlers. Implements custom parsing logic for command interpretation and process management.

Message Oriented TCP (M-TCP)

C Networking UDP

Overview

A reliable, message-oriented communication protocol built on top of unreliable UDP, implementing end-to-end reliability guarantees.

Key Features

  • Message-oriented, in-order delivery over UDP
  • Full-duplex communication support
  • Selective-Repeat ARQ for reliability
  • Flow control and congestion avoidance
  • Multithreading with mutex synchronization
  • Sliding window protocol implementation
  • Packet drop simulation for testing

Technical Implementation

Implements custom packet headers, acknowledgment mechanisms, and timeout-based retransmission. Uses POSIX threads for concurrent send/receive operations.

Cache Coherence Simulator (MSI Protocol)

C++ Systems Programming Computer Architecture

Overview

A multi-core cache coherence simulator that implements the MSI (Modified, Shared, Invalid) protocol to demonstrate how cache coherence is maintained in shared memory systems.

Key Features

  • Full implementation of MSI cache coherence protocol
  • Simulates multi-core processor interactions
  • Tracks cache states (Modified, Shared, Invalid)
  • Monitors cache hits and misses across cores
  • Models shared memory system behavior
  • Validates cache coherence correctness

Technical Implementation

Built with C++ using object-oriented design principles. Implements state machines for cache line transitions, bus snooping protocols, and memory access patterns. Includes comprehensive testing framework and performance metrics.

GPT-based Language Model

Python PyTorch Tiktoken Transformers

Project Timeline

Nov '24 - Dec '24

Objective

Building a GPT-2 based architecture for dialogue generation using Little Shakespeare dataset.

Key Achievements

  • Designed and implemented a 124M parameter GPT-based language model trained on 300k+ tokens from Little Shakespeare dataset
  • Built custom MLP and multi-head attention mechanisms from scratch
  • Utilized gradient accumulation for handling large batch sizes efficiently
  • Implemented mixed-precision training to reduce memory usage and improve computational efficiency
  • Applied gradient clipping and learning rate scheduling for stable training
  • Fine-tuned the pre-trained model on SQuAD dataset using Hugging Face Datasets library
  • Achieved consistent loss reduction through hyperparameter tuning and advanced optimizer settings
  • Successfully adapted the model for question answering tasks on SQuAD

Technical Implementation

Implemented a full GPT-2 architecture including positional encodings, layer normalization, and causal masking. Used PyTorch for model development and Tiktoken for tokenization. Applied modern training techniques including:

  • Mixed-precision training with automatic mixed precision (AMP)
  • Gradient accumulation for effective large batch training
  • Learning rate warmup and cosine decay scheduling
  • Gradient clipping for training stability

Results

Achieved significant training efficiency improvements and demonstrated strong performance on both dialogue generation and question answering tasks.
